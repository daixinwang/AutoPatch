# Apply Patch → Create GitHub PR 设计文档

**日期：** 2026-05-09
**范围：** AutoPatch — Agent 修复 issue 后，用户手动触发创建 GitHub Pull Request

---

## 背景

Agent 完成 issue 修复后，前端展示 diff 内容，目前仅提供一行提示文字 `git apply issue-{N}.diff`，没有实际的 apply 操作。本功能在结果区增加 "Create PR" 按钮，用户检查 diff 后点击，后端自动将补丁应用到目标仓库并创建 PR。

---

## 约束

- Pipeline 成功完成后 `workspace`（tmp_dir）已被清理，无法复用。
- 前端已拥有 `diffContent`（result 事件）、`repoUrl` 和 `issueNumber`（lastInput）。
- GITHUB_TOKEN 需对目标仓库有 write 权限（`contents: write`, `pull-requests: write`）。

---

## 方案选择

选定**方案一：后端新端点 + 重新 clone**：

- 最可靠，正确处理所有 diff 场景（新文件、删除、二进制）
- 复用现有 `github_client.py` 架构
- 前端无需持久化 taskId 或 diff 路径

---

## 整体流程

```
前端 ResultArea
  └─ [Create PR] 按钮点击
       │
       ▼
POST /api/apply
  { repoUrl, issueNumber, diffContent }
       │
       ▼  后端
  1. parse_github_url()        → RepoInfo
  2. fetch_repo_metadata()     → default_branch
  3. RepoWorkspace.clone()     → tmp_dir
  4. git checkout -b autopatch/issue-{N}
  5. git apply --whitespace=fix < diffContent
  6. git add -A && git commit
  7. git push https://{token}@github.com/{owner}/{repo}.git {branch}
  8. GitHubClient.create_pull_request()
  9. workspace.cleanup()
       │
       ▼
  { prUrl, branchName }
       │
       ▼
前端: 按钮变为 "View PR ↗" 外链
```

## API 设计

### Request

```
POST /api/apply
Authorization: Bearer {AUTOPATCH_API_KEY}   # 与其他端点一致
Content-Type: application/json

{
  "repoUrl":     "owner/repo",   // 支持与 /api/patch 相同的所有格式
  "issueNumber": 42,
  "diffContent": "diff --git a/..."
}
```

### Response（成功）

```json
{
  "prUrl":      "https://github.com/owner/repo/pull/7",
  "branchName": "autopatch/issue-42"
}
```

### Response（失败）

```json
{ "detail": "人类可读的错误信息" }
```

## 后端实现

### `github_client.py` — 新增方法

```python
def create_pull_request(
    self,
    repo_info: RepoInfo,
    head_branch: str,
    base_branch: str,
    title: str,
    body: str,
) -> str:
    """调用 GitHub API 创建 PR，返回 PR HTML URL。"""
    url = f"{repo_info.api_base}/pulls"
    resp = self._session.post(url, json={
        "title": title,
        "body":  body,
        "head":  head_branch,
        "base":  base_branch,
    }, timeout=REQUEST_TIMEOUT)
    resp.raise_for_status()
    return resp.json()["html_url"]
```

### `server.py` — 新增 Pydantic 模型

```python
class ApplyRequest(BaseModel):
    repoUrl:     str
    issueNumber: int
    diffContent: str
```

### `server.py` — `_git_apply_and_push()`（同步，在 executor 执行）

步骤：
1. `git checkout -b {branch}`
2. 将 `diffContent` 写入临时文件，执行 `git apply --whitespace=fix {tmpfile}`
3. `git config user.email "autopatch@bot"` + `git config user.name "AutoPatch"`
4. `git add -A && git commit -m "fix(#{N}): AutoPatch generated patch"`
5. `git push https://{token}@github.com/{owner}/{repo}.git {branch}`

分支冲突处理：若 push 因远端分支已存在而失败，追加时间戳后缀（`autopatch/issue-42-20260509`）重试一次。

### `server.py` — `POST /api/apply` endpoint

```python
@fastapi_app.post("/api/apply", dependencies=[Depends(_verify_api_key)])
async def apply_endpoint(req: ApplyRequest):
    repo_info = parse_github_url(req.repoUrl)
    client    = GitHubClient()
    meta      = client.fetch_repo_metadata(repo_info)
    workspace = RepoWorkspace(repo_info=repo_info)   # 自动创建临时目录
    tmp_dir   = workspace.clone()                    # 返回 Path
    branch    = f"autopatch/issue-{req.issueNumber}"
    token     = os.getenv("GITHUB_TOKEN", "")
    try:
        loop = asyncio.get_running_loop()
        await loop.run_in_executor(
            None, _git_apply_and_push,
            tmp_dir, branch, req.diffContent, repo_info, token,
        )
        pr_url = client.create_pull_request(
            repo_info, branch, meta["default_branch"],
            title=f"[AutoPatch] Fix issue #{req.issueNumber}",
            body=(
                f"This PR was automatically generated by AutoPatch.\n\n"
                f"Closes #{req.issueNumber}"
            ),
        )
        return {"prUrl": pr_url, "branchName": branch}
    except subprocess.CalledProcessError as e:
        raise HTTPException(status_code=422, detail=f"git 操作失败: {e.stderr}")
    except requests.HTTPError as e:
        raise HTTPException(status_code=422, detail=f"GitHub API 错误: {e}")
    finally:
        workspace.cleanup()
```

## 错误处理

| 场景 | 检测方式 | HTTP | 前端提示 |
|---|---|---|---|
| GITHUB_TOKEN 未设置 | 启动时或调用时检查 | 422 | "未配置 GITHUB_TOKEN" |
| Token 无 write 权限 | GitHub API 返回 403 | 422 | "GitHub Token 无写权限，请检查 token scope" |
| diff apply 冲突 | `git apply` 非零退出 | 422 | "Patch 无法直接应用，请手动 git apply" |
| 分支已存在 | push 失败，自动加时间戳后缀重试 | — | — |
| 网络错误 | requests 异常 | 422 | "网络错误，请稍后重试" |

## 前端实现

### `ResultArea.tsx` — 按钮状态机

```
idle      → [Create PR]（GitPullRequest 图标，与 Copy Diff 同行）
creating  → [Creating…]（spinner，禁用）
success   → [View PR ↗]（绿色外链，点击跳转）
error     → [Create PR]（按钮下方行内红色错误文字）
```

### 状态管理（内联，无需新 hook）

```typescript
type PRStatus = 'idle' | 'creating' | 'success' | 'error'
const [prStatus, setPrStatus] = useState<PRStatus>('idle')
const [prUrl,    setPrUrl]    = useState<string>('')
const [prError,  setPrError]  = useState<string>('')

async function handleCreatePR() {
  setPrStatus('creating')
  try {
    const res = await fetch('/api/apply', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        repoUrl:     repoUrl,
        issueNumber: Number(issue),
        diffContent: result.diffContent,
      }),
    })
    if (!res.ok) {
      const { detail } = await res.json()
      throw new Error(detail)
    }
    const { prUrl } = await res.json()
    setPrUrl(prUrl)
    setPrStatus('success')
  } catch (e: any) {
    setPrError(e.message)
    setPrStatus('error')
  }
}
```

## 不在本次范围内

- PR 模板自定义（标题/body 格式固定）
- 多次点击防重（success 状态下按钮已变为外链，自然防止重复创建）
- PR 审查人/label 自动指派
