# Codebase Cleanup & Restructure Design

**Date:** 2026-05-12
**Scope:** Delete dead code, create `core/` package for library modules

---

## Problem

The project root contains a mix of entry-point scripts and library modules, plus two dead files:

- `calc.py` — artifact generated by the agent during a test run (`main.py` TEST_ISSUE). Not real project code.
- `main.py` — early development test harness with a hardcoded issue. The real CLI entry is `autopatch.py`.
- Five library modules (`config.py`, `logging_config.py`, `diff_generator.py`, `github_client.py`, `task_store.py`) sit alongside entry scripts at root level, making it hard to distinguish "what to run" from "what to import".

## Goal

A root directory that contains only entry points and configuration, with library modules in a dedicated `core/` package.

## Target Structure

```
AutoPatch/
├── autopatch.py           # CLI entry point (unchanged)
├── server.py              # FastAPI web server entry (unchanged)
├── run_eval.py            # Eval runner entry (unchanged)
├── core/                  # ← new package
│   ├── __init__.py
│   ├── config.py          # moved from root
│   ├── logging_config.py  # moved from root
│   ├── diff_generator.py  # moved from root
│   ├── github_client.py   # moved from root
│   └── task_store.py      # moved from root
├── agent/                 # unchanged
├── tools/                 # unchanged
├── eval/                  # unchanged
├── frontend/              # unchanged
└── tests/                 # unchanged
```

## Changes

### Delete
- `calc.py`
- `main.py`

### Move to `core/`
- `config.py` → `core/config.py`
- `logging_config.py` → `core/logging_config.py`
- `diff_generator.py` → `core/diff_generator.py`
- `github_client.py` → `core/github_client.py`
- `task_store.py` → `core/task_store.py`

### Update imports (~13 changes across 7 files)

| File | Change |
|------|--------|
| `autopatch.py` | `from config/logging_config/github_client/diff_generator` → `from core.*` |
| `server.py` | `from logging_config/github_client/diff_generator/task_store/config` → `from core.*` |
| `agent/graph.py` | `from config` → `from core.config` |
| `core/github_client.py` | `from config` → `from .config` (relative import, same package) |
| `eval/evaluator.py` | `from diff_generator` → `from core.diff_generator` (dynamic import in function) |
| `tests/test_github_client.py` | `from github_client` → `from core.github_client` |
| `tests/test_task_store.py` | `from task_store` → `from core.task_store` |

## Out of Scope

- `agent/` and `tools/` restructuring — they are already organized; moving them would add churn with no immediate benefit.
- Renaming any files beyond the moves listed above.
- Changes to `eval/` internal structure.

## Verification

After the refactor, run the test suite to confirm nothing is broken:

```bash
python -m pytest tests/ -q
```

All tests must pass before the change is considered complete.
